

Design of Qt based Editor
-------------------------

- Editor Classes
	- node creator library : optionaly divided by categories (look at diagramscene)
	- main view
	- object list: ripresa da motore
	- object properties: ripresa da motore e basata su qpropertywidget
	- optionale: console window sotto
	
- Diagram Classes
	- nodeblock: base class or just the only class for a single block
	- nodeport: tipizzato sul tipo e con semantica: ingresso o uscita, e numerosity: single o multiple
	- nodeconnection

- Semantica CoCo
	
	- Plugin: contenitore di TaskTemplate
		scansione di un folder per estrarre i Plugin dalle DLL e ciascuno crea un o più TaskTemplate
	- TaskTemplate
		nome
		attributi
		porte
	- TaskInstance: associata a un TaskTemplate e agganciata al NodeBlock per la parte grafica
		ogni porta del TaskInstance è agganciata al NodePort

	- TaskConnection: connette due TaskInstance e agganciato al NodeConnection

	Semplificazione: TaskInstance == NodeBlock

	- Loading da <-> verso XML	
		crea TaskInstance e TaskConnection

- Special Handling of Peer
	ogni component ha un NodePort father single input, e children sotto multiple output

Eventi:

	NodeBlock selezionato -> Object Properties viewer mostra il Property del TaskInstance associato

Peculiarità: tipi delle connessioni e degli attributi
